The **Domain Operations** section is populated from the domain knowledge interview — operational commands, workflows, and project-specific procedures the agent should follow.

The **Delegation** section documents when and how to use each delegation preset, spec file requirements, and the decision framework for choosing presets. Only included if delegation was set up.

Example delegation section:
```markdown
## Delegation

### When to Delegate
- Bug fixes with clear specs → `fix-commit` preset
- Research/analysis tasks → `analyze` preset
- Features requiring review → `fix` preset
- Autonomous feature work → `full` preset

### Spec File Requirements
Every delegated task needs a spec file with:
- **Problem**: What's wrong or what's needed
- **Expected behavior**: What success looks like
- **Acceptance criteria**: Testable conditions
- **Constraints**: Any limitations or rules

### Decision Framework
| Task Type | Preset | Why |
|-----------|--------|-----|
| Code review / research | analyze | Read-only, safe |
| Bug fix (you review) | fix | No git, you control commits |
| Bug fix (auto-commit) | fix-commit | TDD + auto-commit |
| Feature (full autonomy) | full | Everything except destructive ops |
```

The **Delegation Tooling** section documents the claw script usage, preset table, examples, and key requirements. Only included if delegation was set up.

Example delegation tooling section:
```markdown
## Delegation Tooling

### Usage
```bash
./claw run <preset> <prompt>        # Inline prompt
./claw run <preset> -f <spec-file>  # Spec-driven
./claw presets                       # List presets
./claw worktrees                     # List active worktrees
./claw clean                         # Remove clean worktrees
```

### Presets
| Preset | Permissions | Use Case |
|--------|------------|----------|
| analyze | Read-only | Research, code review |
| fix | Edit + test | Bug fixes (you review diff) |
| fix-commit | Edit + test + git | Auto-commits, TDD |
| full | All except destructive | Full autonomy |

### Key Requirements
- Commit spec files to main before running (worktrees branch from HEAD)
- Child instances clear CLAUDECODE= and ANTHROPIC_API_KEY= to prevent nesting
- Each worktree gets an isolated git branch
```
